# WeThink Interior Design — Official Site

Next.js 16 (static export) + React 19 + Tailwind CSS v4 + Motion, deployed on GitHub Pages.

## Development

```bash
npm install
npm run images   # Run on first setup or after adding/changing portfolio images: generates optimized images and the manifest
npm run dev      # Local development at http://localhost:3000
```

## Build

```bash
npm run images   # Ensure public/portfolio and src/portfolio-manifest.json are up to date
npm run build    # Static export to out/
```

> `public/portfolio/` and `src/portfolio-manifest.json` are generated by `npm run images`.
> After changing source images in `assets/portfolio/` or `src/config/portfolio.config.js`, always re-run `npm run images`.

## Adding a Portfolio Project

1. Place source images in `assets/portfolio/<id>/`, named with two-digit zero padding (`01.jpeg`, `02.jpeg`, …); `01` is the cover.
   Supported source formats: `.jpeg/.jpg/.png/.webp`; recommended long edge ≥ 2000px.
2. Add an entry `{ id, slug, title, subtitle, category }` in `src/config/portfolio.config.js`.
   - `slug`: lowercase hyphenated English, used as the URL `/portfolio/<slug>`; falls back to `id` if omitted.
   - `category`: `residential` / `commercial` / `office`.
3. Run `npm run images`, then `npm run build`; the project detail page is generated automatically.

See `docs/superpowers/specs/2026-06-13-site-redesign-design.md` (Appendix A) for details.

## Image Optimization Strategy

GitHub Pages is a static deployment, so Next.js's built-in image optimization does not work; a build-time pipeline is used instead:

- `scripts/optimize-images.mjs` (sharp): converts source images in `assets/portfolio` into multi-size AVIF/WebP variants plus blurred preview placeholders, and writes `src/portfolio-manifest.json`.
- `src/components/ui/OptimizedImage.jsx`: outputs `<picture>` + `srcset` + blur placeholder + lazy load.
- 132MB of source images (`assets/`, not deployed) → ~14MB of optimized variants (`public/`, deployed).
